1. Executive Summary

India is choking. Air pollution kills 1.67 million Indians every year (Lancet Planetary Health, 2024), making it the nation's deadliest environmental crisis. Delhi's average AQI hovers at 218 — "Very Poor" on the national scale. Yet only 31% of Indian cities have air quality response protocols (CAG Audit, 2024). The data exists. The intelligence layer to act on it does not.

Pavan (Sanskrit: "wind" / "breath of life") bridges this gap. It is a multi-agent AI platform that fuses 5 real-time data sources through a 6-agent intelligence pipeline, transforming raw monitoring data into actionable source attribution, hyperlocal forecasts, enforcement recommendations, and multilingual citizen alerts — in under 3 seconds.

Pavan is not a dashboard. It is an autonomous intelligence system that can attribute pollution to specific source categories with confidence scores, forecast AQI at hyperlocal resolution 72 hours ahead, generate priority-ranked enforcement recommendations compliant with the GRAP framework, run counterfactual intervention simulations ("what if we ban trucks?"), and broadcast health advisories in English, Hindi, Tamil, and Bengali across Telegram, WhatsApp, email, and web channels.

Our Thesis

Multi-agent AI can bridge the gap between environmental monitoring and urban intervention. By orchestrating specialized AI agents that reason over fused data sources — satellite imagery, ground sensors, weather, traffic, and atmospheric models — we can compress the signal-to-intervention cycle from hours/days to seconds.

The Intelligence Gap

India has invested heavily in monitoring infrastructure. The CPCB's CAAQMS network spans 900+ stations with 15-minute pollutant readings. Satellite systems like Sentinel-5P provide daily tropospheric column density maps. MODIS/VIIRS detects active fires in near-real-time. The data exists.

What does not exist is the intelligence layer that can:

  • Attribute — decompose observed pollution into source categories (vehicular, industrial, construction, burning, background) with confidence scores
  • Predict — forecast AQI at hyperlocal (station-level) resolution beyond simple persistence models
  • Recommend — generate priority-ranked, zone-specific enforcement actions grounded in GRAP compliance requirements
  • Simulate — model counterfactual scenarios to estimate the AQI impact of proposed interventions before deployment
  • Alert — reach citizens in their language, on their platform, with actionable health guidance

Why Dashboards Are Not Enough

Current air quality platforms (SAMEER, AirVisual, SAFAR) display data. They show you the AQI is 350. They do not tell you why it is 350 (source attribution), when it will change (forecasting), what should be done (enforcement), or how much a proposed action would help (simulation). Municipal officers need decision support, not data visualization. Citizens need health advisories in their language, not English-only web dashboards.

The Problem Statement, verbatim

"The data exists. The intelligence layer to act on it does not."

Pavan's thesis: a multi-agent AI system — where specialized agents reason over fused data sources in a coordinated pipeline — can bridge the gap from monitoring to intervention in under 3 seconds.

Data Fusion Strategy

No single data source is sufficient. Ground sensors provide precise but sparse readings. Satellites provide broad but coarse coverage. Weather determines dispersion. Traffic explains vehicular contribution. Fire detection explains burning events. Pavan's orchestrator fuses all five through its agent mesh, with each agent responsible for validating, enriching, and reasoning over its assigned source.

5. AI Agents Deep Dive

Pavan's intelligence core is a 6+1 agent system built on a shared BaseAgent abstract class. Each agent produces an AgentResult containing structured data, natural-language reasoning, and a confidence score (0.0 to 1.0). The Orchestrator coordinates their execution in a 4-phase pipeline with parallel execution using Python's asyncio.gather.

Orchestrator Coordinator

File: agents/orchestrator.py • Confidence: N/A (coordinator)
  • Role: Coordinates the entire 4-phase pipeline, manages parallel execution, passes data between agents, merges results, tracks timing
  • Phase 0: Prefetches satellite (Sentinel-5P) and traffic data in parallel
  • Phase 1: Runs Sensor Agent + Weather Agent concurrently via asyncio.gather
  • Phase 2: Runs Anomaly Agent + Attribution Agent concurrently (receives Phase 1 output)
  • Phase 3: Runs Enforcement Agent sequentially (requires Phase 2 results)
  • Summary: Builds executive summary with GRAP stage detection, urgency classification, headline generation
  • Error handling: Failed agents return confidence=0.0; pipeline continues without them
  • Output: Complete analysis with per-agent results, run log, timing metrics, data source inventory

Sensor Agent Data Collection

File: agents/sensor_agent.py • Confidence: 0.95 (real data) / 0.5 (synthetic fallback)
  • Role: Pulls real-time CPCB CAAQMS readings via the CPCB service module
  • Validation: Checks for missing/null pollutant readings, validates AQI range (0-500), flags stale timestamps
  • Hotspot detection: Identifies stations with AQI > 200 as hotspots, with separate thresholds for PM2.5 and PM10
  • Synthetic fallback: When CPCB API is unreachable, generates realistic synthetic readings with diurnal and spatial patterns (confidence drops to 0.5)
  • Output: Station readings array, average/max/min AQI, station count, hotspot list with coordinates

Weather Agent Meteorological Analysis

File: agents/weather_agent.py • Confidence: 0.90
  • Role: Fetches current weather + forecast from OpenWeatherMap, performs atmospheric stability analysis
  • Stagnation detection: Wind speed < 2 m/s indicates atmospheric stagnation (poor dispersion)
  • Inversion risk: Detected when humidity > 70% + calm nighttime conditions trap pollutants near surface
  • Wind shift detection: Flags wind direction spread > 120 degrees (variable conditions)
  • Pollution outlook: Classifies as improving/worsening/stable based on wind forecast trends
  • Output: Current conditions, wind analysis (speed, direction, stagnation, dispersion quality), forecast array, alerts

Anomaly Agent Spike Detection

File: agents/anomaly_agent.py • Confidence: 0.75
  • Role: Detects abnormal pollution readings through statistical methods
  • Method 1 — Spatial Z-score: Computes each station's deviation from the citywide mean. Z > 2.0 = anomaly; Z > 3.0 = critical
  • Method 2 — PM2.5/PM10 ratio: Ratio > 0.8 flags combustion sources (fine particulate dominance); ratio < 0.4 flags construction dust
  • Cause guessing: Cross-references detected anomalies with known industrial hotspots and time-of-day patterns
  • Output: Anomaly list with station, severity (warning/critical), z-score, probable cause, coordinates

Attribution Agent Source Decomposition

File: agents/attribution_agent.py • Confidence: 0.65
  • Role: Decomposes pollution at each station into 5 source categories: vehicular, industrial, construction, burning, background
  • Land use profiles: 5 Delhi zones (north/south/east/west/central) with distinct industrial/residential/commercial/green/transport ratios
  • Satellite cross-reference: MODIS fire proximity (<0.5 degrees) boosts burning attribution by 2.5x for crop burning, 1.5x for industrial fires; high-confidence fires (>80%) get additional 1.3x boost
  • Temporal adjustments: Rush hour (7-10AM, 5-9PM) boosts vehicular 1.8x; burning hours (5-7AM, 7-10PM) boost burning 3.0x; work hours boost construction 1.5x
  • PM ratio correction: PM2.5/PM10 > 0.7 boosts combustion sources; < 0.4 boosts construction dust
  • Wind adjustment: Stagnation (<2 m/s) amplifies local vehicular/burning, reduces distant industrial
  • Normalization: All source fractions sum to 1.0 with per-source confidence scores
  • Output: Per-station attribution with sources, confidence map, dominant source, city-wide average

Enforcement Agent Action Generation

File: agents/enforcement_agent.py • Confidence: 0.80
  • Role: Generates priority-ranked enforcement recommendations based on anomaly and attribution analysis
  • 3-tier recommendation system:
  • Tier 1 (Anomaly-based): Deploy inspection teams to critical anomaly locations
  • Tier 2 (Attribution-based): Industrial > 30% triggers factory inspection; construction > 20% triggers site audit; burning > 15% triggers patrol dispatch
  • Tier 3 (Hotspot-based): Top 3 stations with AQI > 250 get zone-specific recommendations
  • Output: Up to 10 priority-ranked recommendations with action, location, justification, urgency level

Citizen Agent Public Communication

File: agents/citizen_agent.py • Confidence: 0.85
  • Role: Generates multilingual health advisories across 5 severity levels in 4 languages
  • Languages: English (EN), Hindi (HI), Tamil (TA), Bengali (BN)
  • Severity levels: Good, Moderate, Poor, Very Poor, Severe — each with language-specific health guidance
  • Zone alerts: Per-station alerts for any station in "poor" or worse category
  • WhatsApp formatting: Emoji-coded messages with bold text, station-specific hotspot list, Pavan branding
  • Output: City-wide advisory in 4 languages, zone alert list, WhatsApp-formatted messages (EN + HI)

6.2 Gaussian Plume Atmospheric Dispersion

Pavan implements a simplified Gaussian plume model for atmospheric dispersion prediction, used by the Attribution Agent to weight source contributions based on wind direction and atmospheric stability.

C(x,y) =  Q⁄2πuσyσz × exp(-y2/2σy2) × [exp(-(z-H)2/2σz2) + exp(-(z+H)2/2σz2)]
Gaussian Plume Concentration Formula (ground-level reflection term included)

Where: C = concentration (ug/m3), Q = emission rate (ug/s), u = wind speed (m/s), σy, σz = horizontal and vertical dispersion coefficients, H = effective stack height (m), x = downwind distance, y = crosswind distance.

Implementation

# Dispersion coefficients with distance correction
sigma_y = sc["sigma_y"] * x * (1 + 0.0001 * x) ** (-0.5)
sigma_z = sc["sigma_z"] * x * (1 + 0.0001 * x) ** (-0.5)

# Stability classification based on wind speed, hour, cloud cover
def classify_stability(wind_speed, hour, cloud_cover=50):
    is_day = 6 <= hour <= 18
    if is_day:
        if wind_speed < 2: return "A" if cloud_cover < 50 else "B"
        elif wind_speed < 5: return "B" if cloud_cover < 50 else "C"
        else: return "C" if cloud_cover < 50 else "D"
    else:
        if wind_speed < 3: return "F"
        elif wind_speed < 5: return "E"
        else: return "D"

10. Innovation Highlights

Pavan is not an incremental improvement on existing air quality dashboards. It introduces seven architectural and functional innovations that, combined, represent a new paradigm for environmental intelligence platforms.

#InnovationWhat Exists TodayWhat Pavan Does Differently
1 Multi-Agent Coordination Monolithic applications with sequential processing 6 specialized agents running in a 4-phase pipeline with parallel execution, confidence scoring, and error-tolerant orchestration. Each agent reasons independently and contributes its expertise to a merged analysis.
2 Counterfactual Simulation Historical analysis only ("what happened") "What if we ban trucks?" — models 5 intervention types with per-station granularity, source-attribution-weighted reduction factors, and historical precedent from published Delhi studies.
3 5-Source Data Fusion Single-source (ground sensors only) Fuses CPCB ground sensors + Sentinel-5P satellite NO2 + MODIS fire detection + weather + traffic mobility data through specialized agents, each validating and enriching its assigned source.
4 Atmospheric Dispersion Modeling Simple averaging or IDW interpolation Gaussian plume model with Pasquill-Gifford stability classes (A-F), wind-direction-aware source attribution, and 15x15 dispersion grids per source.
5 4-Channel Multilingual Alerts English-only web dashboards 4 languages (EN/HI/TA/BN) across 4 channels (Telegram/WhatsApp/Email/Web chatbot) with severity-adapted health guidance including N95 advice, school closure triggers, and emergency contacts.
6 GRAP Compliance Automation Manual stage determination by officials Automatic GRAP stage detection (I-IV) with 28 required actions, one-click compliance report generation, and station-level violation counts.
7 Source Attribution with Confidence No attribution or static averages Dynamic decomposition into 5 sources with per-source confidence scores, cross-referenced against satellite fire data, traffic patterns, PM ratios, wind, and time-of-day.

11. Business Impact

Lives at Stake

With 1.67 million air pollution deaths annually in India, even a 1% reduction through better-targeted interventions would save 16,700 lives per year. Pavan's contribution is not a cure — it is a decision support system that helps municipal officers deploy resources where they matter most.

Municipal Decision Support

  • Where to deploy inspectors: The Enforcement Agent's priority-ranked recommendations tell officers which zones need attention first, based on anomaly severity and source attribution
  • When to escalate: Automatic GRAP stage detection eliminates manual AQI monitoring delays
  • What to do: The Intervention Simulator lets officers compare the projected impact of different actions before committing resources
  • How to communicate: Multi-language citizen alerts reach non-English-speaking populations who are often the most vulnerable

Architecture Supports Horizontal Scaling

  • City-agnostic: All agents accept a city parameter. Adding a new city requires zero code changes — only a CPCB station mapping
  • Data source pluggable: New data sources implement a simple service interface and register with the Orchestrator's Phase 0
  • Agent mesh extensible: New agents extend BaseAgent with run(query, context) → AgentResult. The Orchestrator discovers them at startup
  • Cloud-native deployment: Vercel edge CDN (26 global PoPs), Railway Docker containers (auto-scaling), Supabase managed Postgres (connection pooling, pgvector-ready)
  • Stateless backend: FastAPI backend is stateless — horizontal scaling via Railway container replicas requires no code changes
